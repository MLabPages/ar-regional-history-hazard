# AR地域理解・歴史・防災統合プラットフォーム MVPプロトタイプ

スマートフォンのカメラ映像に「過去の古写真」「地域の変遷」「防災・ハザードリスク」を一つのAR空間として重ね合わせて表示する統合型WebARプロトタイプです。

## 1. アプリの主要機能

1. **① 歴史・観光レイヤー (History)**
   - 現在地周辺の歴史スポットをARピンとして浮遊表示
   - モーダルから「古写真と現実の景観を比較」を選択すると、ARカメラ画面上で古写真の透過率スライダー（0%〜100%）を使ったリアルタイム重ね合わせ比較が可能

2. **② 地域理解・まち歩きレイヤー (Community)**
   - かつての街道跡（旧京街道）、産業遺構、旧駅跡など「土地の記憶」を可視化

3. **③ 防災・ハザードレイヤー (Disaster)**
   - 国土交通省・国土地理院の公式ハザードタイルを重ねて表示（地点別の深さは公式凡例で確認）
   - 避難所データは公式確認前のため、具体的な施設・方角・距離の誘導を停止

4. **位置・方角シミュレーター (室内・PC検証用)**
   - 実地フィールドワークだけでなく、PCブラウザや室内でも位置プリセットや方位角度（0〜360°）を模擬コントロール可能

## 2. 開発・検証の起動手順

本アプリは単体Webアプリ（HTML5/JS）として構築されています。
ローカルのWebサーバー（例: Python http.server や Node.js npx serve / Live Server 等）で起動してブラウザからアクセスしてください。

```bash
# Pythonの場合
python -m http.server 8080

# または Node.js の場合
npx serve .
```

ブラウザで `http://localhost:8080` または `http://<PCのIP>:8080` (モバイル実機の場合) にアクセスします。
※ スマートフォン実機でカメラや位置情報センサーを使用する場合は、HTTPS接続（または ngrok / Cloudflare Tunnel 等）を推奨します。

## 4. 資料の正確性・出典管理

年代別航空写真は、国土地理院の公式タイル一覧で確認した定義だけを使用します。

* 1928年頃、1936〜1942年頃、1945〜1950年頃、1961〜1969年頃、1974〜1978年頃、現代最新写真
* タイルが取得できない地域・ズームでは「データがありません」と表示し、別年代や現代地図へ黙って切り替えません
* 大阪向けの明治・江戸・平安の位置精度付きXYZタイルは未収録です

江戸期資料はXYZタイルではなく、NDLの資料ページとIIIFマニフェストを持つ歴史資料として扱います。現在の確認済み候補は「浪花名所図会 八けん屋着船之図」「浪花名所図会 道とんぼりの図」「大坂大繪圖」の3点です。名所絵・絵図は測量図ではないため、概略位置の参考資料として表示します。

NDLの個別IIIF画像リソースIDはHTTP検証が完了するまで本番画面に埋め込みません。資料ページとマニフェストへのリンクを表示し、画像検証済みの資料だけを古写真比較に使用します。

土砂災害（急傾斜地の崩壊）は、公式オープンデータ一覧で確認した大阪府コード27、ズーム2〜17のタイルを使用します。避難所の施設名・座標・対象災害・種別は未確認のため、避難誘導には使用しません。

画面に表示する画像・記述には、資料種別、出典URL、ライセンス、利用状況、位置精度、確認日を持たせています。確認できていない画像は「開発用プレースホルダー」、確認できていない記述は「未確認」と表示します。

## 5. ネイティブ（Flutter / Unity）への拡張ロードマップ

* **Phase 1（現在）**: WebARプロトタイプによる体験検証・オープンデータ整合
* **Phase 2**: Flutter + ARKit/ARCore によるネイティブアプリ化（永続キャッシュ・プッシュ通知）
* **Phase 3**: Unity + VPS (Visual Positioning System) 導入による高度な3D建物の位置合わせとAI古写真復元
